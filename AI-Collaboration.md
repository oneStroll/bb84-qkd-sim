## 第一轮：物理机制到代码的映射

**我的提问**：BB84 里 Alice 用量子态编码比特、Bob 测量、Eve 窃听，这些在普通 Python 代码里要怎么模拟？我不想引入 qiskit 那种重量级框架。

**AI 的分析**：
- 提出用 `(bit, basis)` 元组表示量子态，basis ∈ {Z, X}
- Z 基下：bit 0 → |0⟩，bit 1 → |1⟩
- X 基下：bit 0 → |+⟩ = (|0⟩ + |1⟩)/√2，bit 1 → |-⟩ = (|0⟩ - |1⟩)/√2
- 测量模拟：基一致时结果确定，不一致时用 `np.random.randint(0, 2)` 模拟 50% 随机坍缩

**我的追问**：这样直接用随机数代替量子测量，物理上合理吗？会不会跟真实的量子力学有出入？

**AI 的回应**：在 BB84 中我们只关心测量结果的统计分布，不需要完整量子态演化，所以经典概率模拟是合理的：
- 基一致时测量结果确定 → 对应 Born 规则中投影概率为 1
- 基不一致时各 50% → 对应 |⟨0|+⟩|² = |⟨1|+⟩|² = 1/2
- 这种简化在"只关心测量结果统计特性"的场景下严格成立

**我的判断**：解释说得通，BB84 教学仿真的目的就是验证协议流程和 QBER，不需要完整量子态演化。按这个方案做。

## 第二轮：Eve 的截获-重发攻击与 QBER 推导

**我的提问**：有 Eve 的时候 QBER 为什么会到 25%？我想从概率上搞清楚这个数字怎么来的。

**AI 的推导**：对于 Alice 和 Bob 基一致的位置 i：
- Eve 选对基的概率是 1/2，选对不引入错误
- Eve 选错基的概率是 1/2，选错之后 Bob 有 1/2 概率拿错比特
- 所以 QBER = 1/2 × 1/2 = 1/4

**我的追问**：有没有比截获-重发更厉害的 Eve？如果用集体攻击 QBER 会变吗？

**AI 的分析**：截获-重发属于个体攻击（individual attack），还有集体攻击和相干攻击，那些要用纠缠态分析，复杂度高很多。对于课程作业要求的"基础仿真"，截获-重发已经够了。更复杂的攻击模型需要纠缠态和量子信息论，超出范围了。

**我的判断**：也是，基础仿真做到截获-重发就可以了。在物理文档里提了一句安全阈值 11% 的事，说明更复杂攻击的情况。

## 第三轮：仿真代码的验证策略

**我的提问**：测试要怎么确保 QBER 算出来是准的？比特数有限的时候结果会有波动吧。

**AI 的建议**：
- 大样本（n=50000）下 QBER 应在 25% 附近窄区间内
- 用固定种子保证结果可复现
- 同时验证无 Eve 场景（QBER ≈ 0%）作为对照

**我的补充**：
- 加了 `test_qber_increases_with_eve`，确保有 Eve 时 QBER 一定高于无 Eve
- 固定种子防止代码改动后物理逻辑被破坏
- 区分统计波动和物理错误，前者可以接受，后者必须排除

**执行结果**：19 个测试全部通过，无 Eve 时 QBER ≈ 0%，有 Eve 时 ≈ 25%，跟理论吻合。

## 第五轮：Wiki 公式渲染问题（简要）

GitHub Wiki 的 MathJax 渲染跟标准 Markdown 有不少差异，来回修了好几轮：

| 问题 | 解决方案 |
|------|---------|
| `$|0\rangle$` 中 `|` 被当表格分隔符 | 源文件用 Unicode `∣` |
| Wiki 对 `$...$` 内联数学支持有限 | 写转换脚本，剥离内联 `$`，保留 `$$...$$` |
| `\#` 报错、`\-` 被当连字符等 | 逐个调整 LaTeX 写法 |

经过多轮"提交-反馈-修正"，最终主仓库保留完整 LaTeX，Wiki 用转换脚本做兼容处理。
