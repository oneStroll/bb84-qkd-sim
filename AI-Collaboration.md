## 第一轮：物理机制到代码的映射

**我的提问**：BB84 协议中，Alice 用 Z 基和 X 基编码量子态，Bob 随机选择测量基进行测量，Eve 的截获-重发会引入扰动。如何在经典计算机上模拟这些量子过程？

**AI 的分析**：
- 提出用 `(bit, basis)` 元组表示量子态，basis ∈ {Z, X}
- Z 基下：bit 0 → |0⟩，bit 1 → |1⟩
- X 基下：bit 0 → |+⟩ = (|0⟩ + |1⟩)/√2，bit 1 → |-⟩ = (|0⟩ - |1⟩)/√2
- 测量模拟：基一致时结果确定，不一致时用 `np.random.randint(0, 2)` 模拟 50% 随机坍缩

**我的追问**：这个模拟本质上是用条件概率代理了量子振幅演化。它的物理依据是什么？

**AI 的回应**：在 BB84 协议中，我们只关心测量结果的统计分布而非量子态的完整演化，因此可以用经典概率模拟替代密度矩阵运算。具体而言：
- 基一致时测量结果确定 → 对应 Born 规则中投影概率为 1
- 基不一致时各 50% → 对应 |⟨0|+⟩|² = |⟨1|+⟩|² = 1/2
- 这一简化在"仅关心测量结果统计特性"的场景下严格成立

**我的判断**：这个抽象是合理的。BB84 教学仿真的目的在于验证协议流程和 QBER，不需要完整量子态演化。批准实现。

## 第二轮：Eve 的截获-重发攻击与 QBER 推导

**我的提问**：Eve 的截获-重发攻击如何定量影响 QBER？需要从条件概率角度给出完整推导。

**AI 的推导**：对于 Alice 和 Bob 基一致的位置 i：
- $P(\gamma_i = \theta_i) = 1/2$：Eve 选对基，不引入误码
- $P(\gamma_i \neq \theta_i) = 1/2$：Eve 选错基，此时 Bob 以 1/2 概率得到错误比特
- $\text{QBER} = P(\gamma_i \neq \theta_i) \times P(\tilde{b}_i \neq b_i \mid \gamma_i \neq \theta_i) = 1/2 \times 1/2 = 1/4$

**我的追问**：这个推导假设了 Eve 对每个量子比特独立攻击。如果 Eve 实施集体攻击（collective attack），QBER 是否会变化？

**AI 的分析**：截获-重发攻击属于个体攻击（individual attack），每个量子比特被独立截获和重发。集体攻击和相干攻击（coherent attack）确实存在更复杂的纠缠资源利用方式，但在本科教学的 BB84 仿真中，截获-重发攻击是最经典的 Eve 模型，也是理解 QKD 安全性的起点。更复杂的攻击模型需要引入纠缠态和量子信息论分析，超出了本仿真的范围。

**我的判断**：对于课程要求的"全链路基础仿真"，截获-重发攻击已足够。在 `docs/physics_background.md` 中补充了安全阈值 11% 的注解，指明更复杂攻击的理论边界。

## 第三轮：仿真代码的验证策略

**我的提问**：如何确保 QBER 的仿真结果具有统计显著性？有限的量子比特数会导致波动。

**AI 的建议**：
- 大样本（n=50000）下 QBER 应在 25% 附近窄区间内
- 用固定种子保证结果可复现
- 同时验证无 Eve 场景（QBER ≈ 0%）作为对照

**我的补充**：
- 增加了 `test_qber_increases_with_eve` 测试，确保有 Eve 时 QBER **总是高于** 无 Eve 时
- 固定种子的测试从单元测试层面保证代码改动不会意外破坏物理逻辑
- 测试中要区分"统计波动"和"物理错误"——前者可接受，后者必须排除

**执行结果**：19 个测试全部通过，验证了理想信道和 Eve 攻击两种场景。

## 第五轮：Wiki 公式渲染问题（简要）

GitHub Wiki 的 MathJax 渲染与标准 Markdown 存在兼容性差异，主要问题及解决方案：

| 问题 | 解决方案 |
|------|---------|
| `$|0\rangle$` 中 `|` 被当表格分隔符 | 源文件使用 Unicode `∣`（U+2223） |
| Wiki 对 `$...$` 内联数学支持有限 | 编写转换脚本，剥离内联 `$`，保留 `$$...$$` |
| `\#` 在 MathJax 中报错 | QBER 公式改为 `|\{i \in T \mid ...\}|` |
| `\-` 被解析为连字符命令 | 矩阵行间加空格：`\\ -1` |
| 部分 Unicode 渲染异常 | 关键公式改用 `$$...$$` 单独成段 |

经过多轮"提交-反馈-修正"循环，最终实现了主仓库（完整 LaTeX）和 Wiki（兼容格式）双轨并行。

## 总结

AI 在本项目中承担了协同研究员的角色：

- **代码层面**：快速搭建仿真框架、编写测试、处理 CI/CD 配置，大幅缩短了从需求到可运行代码的周期
- **物理文档层面**：辅助生成狄拉克符号表述和公式推导，但每一条公式都经过我审查确认
- **问题排查层面**：在 Wiki 渲染等工程问题上通过多轮对话迭代解决

AI 工具的价值在于放大研究者的生产力——将物理直觉快速转化为可执行的代码和规范的文档。但量子力学的基本概念、仿真逻辑的正确性、测试阈值的设定，这些核心判断必须由研究者自己完成。AI 提供候选方案，我决定方向和质量标准。
