# AI 协同日志：量子密钥分发仿真中的"研究员-大模型"协作实践

> 本文件记录在 BB84 QKD 协议仿真项目中，开发者以"协同研究员"的模式引导免费/开源大模型 Claude Code 突破量子信息领域工程难题的全过程。AI 工具在本项目中扮演了代码起草者、物理文档协作者和架构讨论伙伴的角色，但所有关键决策和物理正确性均由开发者审查把关。

## 核心量子领域难题与 AI 协同突破

### 难题一：如何将 BB84 的量子力学图像映射为可执行的仿真代码？

**背景**：BB84 协议涉及量子叠加态、测量坍缩、不可克隆定理等量子力学核心概念，如何在不引入完整量子态演化库（如 qiskit）的前提下，用经典 Python 代码准确模拟这些量子效应？

**协作过程**：
- 开发者提出采用**经典统计模拟**的路线，而非完整的量子态向量演化
- AI 据此设计了 `(bit, basis)` 编码方案，用 `np.random.choice` 模拟 Born 规则中的概率测量
- 开发者审查后要求确保基一致/不一致的物理区分在代码中可见，AI 补充了代码注释中的狄拉克符号对应

**关键突破**：在保持物理准确性的前提下，将量子测量坍缩抽象为条件概率分支——基一致时 `bits[match] = alice_bits[match]`（确定结果），基不一致时 `bits[~match] = np.random.randint(0, 2)`（50% 随机）。这一抽象既保证了 QBER = 25% 的物理预测，又避免了引入量子库带来的依赖复杂性。

**开发者主导的决策**：
- ✅ 采用经典统计模拟路线（否决了 AI 提出的量子态向量方案）
- ✅ 物理准确性优先于代码复杂度
- ✅ 狄拉克符号必须出现在代码注释和文档中

---

### 难题二：如何精确计算并验证 Eve 截获-重发攻击引入的 QBER？

**背景**：QBER = 25% 是 BB84 协议中 Eve 存在与否的核心判据，需要从数学上严谨推导并在代码中可复现。

**协作过程**：
- 开发者首先给出 QBER 的理论推导需求：P(选错基) × P(选错基后出错)
- AI 起草了条件概率表，用 $\frac{1}{2} \times \frac{1}{2} = \frac{1}{4}$ 的形式化语言表述
- 开发者指出需要区分"Eve 选错基"和"Bob 最终出错"两个独立事件，AI 修正了推导表述
- 开发者要求测试必须考虑统计波动（有限样本效应），AI 设计了多层测试策略

**关键突破**：测试设计体现了科研思维——不追求单一测试通过，而是构建统计显著性检验：大样本（n=20000）下 QBER 置信区间收敛到 [20%, 30%]，小样本（n=1000）允许更大波动。这一策略来自开发者的研究方向引导，AI 负责落地为 pytest 代码。

### 难题三：如何在 GitHub Wiki 中正确渲染量子力学公式？

**背景**：物理文档包含大量狄拉克符号（$\vert 0\rangle$、$\langle\psi\vert$ 等），但 GitHub Wiki 的 Markdown/MathJax 渲染与标准 GitHub Markdown 存在差异，导致管道符 `|` 被误解析为表格分隔符、下划线 `_` 被误解析为斜体标记、反斜杠 `\` 转义层级混乱等工程问题。

**协作过程**：
- 开发者发现问题后逐一反馈（表格断裂、公式报错、内联数学不渲染）
- AI 先后尝试了 `\vert`、`\(...\)`、`\_` 转义、纯文本转换等多种方案
- 每次修复后开发者验证并指出遗留问题，形成"反馈-修正-验证"循环
- 最终确定：源文件使用 Unicode `∣`（U+2223）代替 `|`，配合 Wiki 专用转换脚本

**关键突破**：这个问题的解决体现了工程师思维——不追求单一完美的渲染方案，而是通过数轮"问题报告 → 方案尝试 → 用户验证"的迭代，找到了"源文件兼容主仓库 + 转换脚本适配 Wiki"的双轨策略。

## AI 协同工作流总结

```
开发者（研究主导）                  AI 工具（执行协同）
      │                                    │
      ├─ 提出物理问题/需求 ──────────────→ 提供代码/文档方案
      │                                    │
      ├─ 审查方案正确性 ←────────────── 产出草案
      │                                    │
      ├─ 修正方向/补充约束 ──────────────→ 迭代优化
      │                                    │
      ├─ 终审物理准确性 ←────────────── 提交最终版本
      │                                    │
      └─ 整合到项目 ────────────────── 任务完成
```

## 协作伦理与边界

1. **物理正确性由开发者负责**：AI 生成的狄拉克符号公式、QBER 数学推导均经开发者逐行验证
2. **代码逻辑由开发者把关**：测试阈值（QBER < 2%、置信区间 20%-30%）由开发者根据统计学原理设定
3. **AI 不替代学术诚信**：本日志如实记录 AI 参与范围，所有核心算法设计和物理理解均来自开发者
4. **AI 作为"加速器"而非"替代品"**：AI 将开发者的物理洞察快速转化为可执行代码和规范文档，缩短了从理论到实现的周期
